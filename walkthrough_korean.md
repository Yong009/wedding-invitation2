# 스크롤 문제 해결 워크스루

사용자 환경(특히 데스크톱)에서 발생했던 '스크롤 반응 없음' 및 '초기 점프 현상'을 해결하기 위한 작업 내용입니다.

## 주요 변경 사항

### 1. 스크롤 '데드존' 제거 및 전역 스크롤성 강화 (`src/index.css`)
- **강제 스크롤 활성화**: `html`과 `body`에 `overflow-y: auto !important` 및 `overflow-x: hidden`을 설정하여, 화면상의 어느 곳에서도 스크롤이 자연스럽게 작동하도록 보장했습니다.
- **스크롤바 거터 안정화**: `scrollbar-gutter: stable`을 추가하여 스크롤바가 생기고 사라질 때 화면이 좌우로 흔들리는(Layout Shift) 현상을 방지했습니다.
- **부드러운 스크롤**: `scroll-behavior: smooth`를 적용하여 페이지 내 이동이 자연스럽게 이루어지도록 했습니다.

### 2. 가시성 높은 전용 스크롤바 디자인 (`src/index.css`)
- 일부 브라우저 환경에서 스크롤바가 숨겨져 하단 콘텐츠가 있는 것을 인지하지 못하는 문제를 해결하기 위해, 초대장의 보라색(Lavender) 톤에 맞춘 전용 스크롤바를 제작하여 가시성을 확보했습니다.

### 3. 초기 이미지 로딩 시 레이아웃 시프트 방지 (`src/App.jsx`)
- **메인 이미지 비율 고정**: `intro` 섹션의 메인 이미지가 로딩되기 전에도 빈 공간을 미리 차지하도록 `aspect-ratio`를 설정했습니다. 이로 인해 이미지가 늦게 불러와지면서 페이지가 갑자기 길어지거나 스크롤이 튀는 현상을 근본적으로 해결했습니다.
- **플레이스홀더 배경**: 이미지가 완전히 뜨기 전까지 연한 회색 배경(`background-color`)을 보여주어 사용자에게 로딩 중임을 시각적으로 알려줍니다.

## 결과 확인 방법
1. 데스크톱에서 사이트 접속 후, 마우스를 화면의 가장자리(화면 중앙이 아닌 빈 공간)에 두고 휠을 돌려보세요. 이제 정상적으로 스크롤됩니다.
2. 새로고침 시 페이지 상단 이미지가 로딩될 때 스크롤 바의 유무나 페이지 스크롤 위치가 갑자기 변하지 않는지 확인해 보세요.
